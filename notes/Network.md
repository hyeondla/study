> 패킷

네트워크를 통해 전송되는 데이터의 기본 단위
큰 데이터를 작은 패킷으로 분할

송신 측에서 패킷을 보낼 때 각 패킷에 순서대로 번호를 붙임
수신 측에서는 패킷이 도착하면 번호 순으로 정렬

> 대역폭

최대 전송 속도로 정보를 전송할 수 있는 단위 시간당 전송량

> 비트

정보를 나타내는 최소 단위
비트 정보를 전기 신호로 변환하여 데이터 전송

1 바이트 = 8 비트

> LAN

근거리 통신망
속도 빠름, 오류 적음

유선/무선 방식

> WAN

광역 통신망
속도 느림, 오류 많음

>  DMZ

공개 서버 (웹, 메일, DNS, ...)
외부 네트워크와 내부 외트워크 사이에 위치한 중간 지대 (서브넷) 
외부 공격자가 내부 네트워크에 침입하는 것을 막는 보안 역할

인터넷 서비스 제공자
SK, KT, U+ 통신 회사

> OSI 모델

- 물리 계층
  + 시스템 간의 물리적인 연결
  + 전기 신호의 변환 및 제어
  + 랜 카드 : 컴퓨터의 네트워크 연결 및 데이터 전송
  + 전송 매체 : 데이터가 흐르는 물리적인 선로, 유선/무선
  + 트위스트 페어 케이블 = 랜 케이블
    - UTP (일반적으로 사용) / STP
    - RJ-45 : 커넥터, 랜 케이블 양쪽에 붙어 있음
  + 다이렉트 케이블 
    - 같은 순서로 커넥터에 연결
    - 컴퓨터와 스위치를 연결할 때 사용
  + 크로스 케이블 
    - 1,2번 구리 선을 다른 쪽 3,6번에 연결
    - 컴퓨터 간에 직접 랜 케이블로 연결할 때 사용
  + 리피터 
    - 감쇠된 신호 정형화 및 증폭
    - 일대일 통신
  + 허브
    - 신호 정형화 및 증폭
    - 일대다 통신
    - 특정 포트로부터 받은 데이터를 나머지 모든 포트로 전송 (스스로 판단 불가)
- 데이터 링크 계층
  + 데이터 단위 : 프레임
  + 네트워크 기기 간의 데이터 전송 및 물리 주소 결정
  + 이더넷 : 랜에서 데이터를 정상적으로 주고받기 위한 규칙
  + CSMA/CD
    - 이더넷에서 시점을 늦춰 충돌이 발생하지 않게 하는 방법
    - CS : 데이터를 보내려고 하는 컴퓨터가 케이블에 신호가 흐르고 있는지 확인하는 규칙
    - MA : 케이블에 데이터가 흐르고 있지 않다면 데이터를 송신해도 된다는 규칙
    - CD : 충돌이 발생하고 있는지 확인하는 규칙
  + MAC 주소 : 48비트로 구성된 물리 주소, 유일함
  + 이더넷 헤더 : 목적지 MAC 주소(6바이트) + 출발지 MAC 주소(6바이트) + 유형(2바이트)
  + 이더넷 유형 : 이더넷으로 전송되는 상위 계층 프로토콜의 종류(IPv4, ARP, RARP, SNMP, Ethernet, IPv6)를 나타내는 16진수 번호
  + 트레일러(FCS) : 데이터 전송 도중 오류 발생했는지 확인
  + 목적지 MAC 주소와 자신의 MAC 주소가 다르면 데이터 파기
  + ARP
    - IP 주소 → MAC 주소
    - 목적지 주소를 모르면 네트워크에 브로드캐스트 패킷 요청 (ARP 요청)
    - 지정된 IP 주소를 가진 컴퓨터가 MAC 주소를 응답으로 보냄 (ARP 응답)
    - MAC 주소:IP 주소 매핑 정보를 메모리에 저장 (ARP 테이블)
    - 보존 기간(ARP 캐시) 이후 테이블 데이터 삭제
  + 스위치(레이어2스위치/스위칭 허브) 
    - 내부에 MAC 주소 테이블(포트 번호:연결 컴퓨터 MAC 주소 매핑 정보)이 있음
    - MAC 주소 학습 기능 : 테이블에 출발지 MAC 주소가 등록되어 있지 않으면 등록
    - 플러딩 : 수신 포트 외의 모든 포트에 데이터를 송신
    - MAC 주소 필터링 : MAC 주소를 기준으로 출발지를 선택
  + 전이중 통신 방식
    - 동시에 데이터 송수신
    - 스위치에 연결하는 경우
  + 반이중 통신 방식
    - 번갈아가며 데이터 송수신
    - 허브에 연결하는 경우
  + 이더넷 규격
    - 통신속도 전송방식 - 케이블
    - 10BASE-T : 10Mbps BASEBAND UTP
    - 10BASE5 : 10Mbps BASEBAND 500M동축케이블 
- 네트워크 계층
  + 데이터 단위 : 패킷
  + 다른 네트워크와 통신하기 위한 경로 설정 및 논리 주소 결정
  + 다른 네트워크로 전달하려면 IP주소 필요
  + 라우터 
    - 통신을 위한 장비
    - 네트워크 분할 가능 (스위치/허브는 불가)
    - 다른 네트워크에 데이터를 전송할 때 라우터의 IP 주소를 설정해야함 = 기본 게이트웨이
  + 라우팅 
    - 경로를 선택하고 목적지 IP주소까지 어떤 경로로 데이터를 보낼지 결정하는 것
    - 라우팅 테이블 : 경로 정보 등록된 테이블
    - 라우팅 프로토콜 : 라우터 간에 라우팅 정보를 교환하기 위한 프로토콜, RIP/OSPF/BGP
  + IP 헤더 : 출발지 IP주소, 목적지 IP주소 포함
  + IP 주소
    - 공인 : ISP가 제공, 라우터에 할당
    - 사설 : 랜 안의 컴퓨터에 할당 
    - 2진수 32 비트(IPv4), 128비트(IPv6)로 구성
    - 8비트 단위(옥텟)로 나눠서 10진수로 표시
    - 네트워크 ID : 어떤 네트워크인지 나타냄
    - 호스트 ID : 해당 네트워크의 어느 컴퓨터인지 나타냄
    - A 클래스
      + 대규모
      + 네트워크 8, 호스트 24 비트
      + 1.0.0.0 ~ 127.255.255.255
      + 공인 1.0.0.0 ~ 9.255.255.255, 11.0.0.0 ~ 126.255.255.255
      + 사설 10.0.0.0 ~ 10.255.255.255
    - B 클래스
      + 중형
      + 네트워크 16. 호스트 16 비트
      + 128.0.0.0 ~ 191.255.255.255
      + 공인 128.0.0.0 ~ 172.15.255.255, 172.32.0.0 ~ 191.255.255.255
      + 사설 172.16.0.0 ~ 172.31.255.255
    - C 클래스
      + 소규모
      + 네트워크 24, 호스트 8 비트
      + 192.0.0.0 ~ 223.255.255
      + 공인 192.0.0.0 ~ 192.167.255.255, 192.169.0.0 ~ 223.255.255.255
      + 사설 192.168.0.0 ~ 192.168.255.255
    - D 클래스
      + 멀티캐스트 
    - E 클래스 
      + 연구 및 특수용도
    - 네트워크 주소 
      + 호스트 ID가 0인 주소
      + 전체 네트워크의 대표 주소
    - 브로드캐스트 주소 
      + 호스트 ID가 255인 주소
      + 네트워크 내 모든 장비에 한번에 데이터를 전송하는데 사용되는 전용 IP 주소
  + DHCP : IP 주소 자동 할당 프로토콜
  + 서브넷 
    - 분할된 네트워크
    - 네트워크ID + (서브넷ID + 호스트ID)
  + 서브넷 마스크 : 네트워크ID와 호스트ID를 식별하는 값
    - A 클래스 : 255.0.0.0 = /8
    - B 클래스 : 255.255.0.0 = /16
    - C 클래스 : 255.255.255.0 = /24
- 전송 계층
  + 신뢰할 수 있는 통신 구현
  + 재전송 제어 : 오류 발생, 데이터 손상 시 데이터 재전송 요청
  + 전송된 데이터의 목적지 애플리케이션 식별
  + 대역폭 : 정해진 시간 동안 전송될 수 있는 데이터의 양
  + TCP
    - 연결형 통신 : 신뢰성/정확성, 데이터를 목적지에 문제없이 전달
    - 데이터 단위 : 세그먼트
    - 커넥션 : 가상의 통신로, 연결을 확립하고 데이터 전송
    - 헤더 중 
      + 코드 비트 : SYN(연결 요청), ACK(확인 응답), FIN(종료 요청)
      + 일련번호 : 몇번째 데이터인지 표시, 순서 제어
      + 확인 응답 번호 : 몇번째 데이터를 수신했는지 표시
      + 윈도우 크기 : 버퍼의 한계 크기
      + 출발지 포트 번호 
      + 목적지 포트 번호
    - 3-way 핸드쉐이크 
      + 연결 : SYN → SYN+ACK → ACK
      + 연결 종료 : FIN → ACK, FIN → ACK
  + UDP
    - 비연결형 통신 : 효율성, 데이터를 빠르고 효율적으로 전달
    - 데이터 단위 : 데이터그램
    - 일괄 통신 가능 (브로드캐스트)
  + Well-known 포트
    - SSH : 22
    - SMTP : 25
    - DNS : 53
    - HTTP : 80
    - POP3 : 110
    - HTTPS : 443
- 세션 계층
  + 세션 체결, 통신 방식 결정
- 표현 계층
  + 문자 코드, 압축, 암호화 등 데이터 변환
- 응용 계층
  + 이메일, 파일 전송, 웹 사이트 조회 등 애플리케이션에 대한 서비스 제공
  + 클라이언트 : 서비스를 요청하는 측
  + 서버 : 서비스를 제공하는 측
  + 클라이언트의 요청을 전달하기 위해 통신 대상이 이해할 수 있는 데이터로 변환하여 전송 계층으로 전달
  + 클라이언트 측 애플리케이션과 서버 측 프로그램 간의 통신은 응용 계층의 프로토콜을 사용함
  + 프로토콜
    - HTTP : 웹 사이트
      + WWW : HTML, URL, HTTP
      + 80번 포트
      + 요청 : GET(요청 정보, 파일 이름, 버전 등) 전송
      + 응답 : OK(정상 처리), index, html 전송
      + keepalive : 데이터 교환이 모두 끝날때까지 연결 유지
    - DNS : 이름 해석
      + URL을 IP주소로 변환
      + 도메인 이름 : github.com
      + 호스트(서버) 이름 : www
    - FTP : 파일 전송
      + 20번 포트 : 데이터 전송
      + 21번 포트 : 제어
    - SMTP : 메일 송신
      + 25번 포트
    - POP3 : 메일 수신
      + 110번 포트
      + 사용자 인증(이름, 비밀번호) 필요
  + ping : 목적지 컴퓨터와의 통신 확인, ICMP 프로토콜

데이터 송신 시 각 계층에서 헤더를 붙여 나가는 것
하위 계층으로 전송

- 헤더
  + 전송하는 데 필요한 정보
  + 데이터의 내용이나 성격을 식별 또는 
  + 수신 측에 대한 정보
- 트레일러 : 데이터의 마지막에 추가하는 정보

> 데이터 전달과 처리

- OSI 상위 계층부터 캡슐화
  + 전송 계층에서 TCP 헤더 추가 = 세그먼트
  + 네트워크 계층에서 IP 헤더 추가 = 패킷
  + 데이터 링크 계층에서 이더넷 헤더, 트레일러 추가 = 프레임
  + 물리 계층에서 전기 신호로 변환되어 전송
- 라우터
  + 전기 신호 도착
  + 데이터 링크 계층에서 프레임의 목적지 MAC 주소와 자신의 MAC 주소 비교
  + 주소가 같으면 이더넷 헤더, 트레일러 분리 (역캡슐화)
  + 네트워크 계층에 전달
  + 라우팅 테이블과 목적지 IP 주소 비교
  + 출발지 IP 주소 변경
  + 데이터 링크 계층에 전달
  + 이더넷 헤더, 트레일러 추가
  + 물리 계층에서 전기 신호로 변환되어 전송
- 물리 계층부터 역캡슐화
  + 데이터 링크 계층에서 프레임의 목적지 MAC 주소와 자신의 MAC 주소 비교
  + 주소가 같으면 이더넷 헤더, 트레일러 분리
  + 네트워크 계층에 전달
  + 목적지 IP 주소와 웹 서버 IP 주소가 같은지 확인
  + 주소가 같으면 IP 헤더 분리
  + 전송 계층에 전달
  + 목적지 포트 번호 확인하여 어떤 애플리케이션으로 전달해야 하는지 판단
  + TCP 헤더 분리
  + 응용 계층에 전달
- 정적 라우팅
  + 관리자가 미리 라우팅 테이블에 경로를 수동으로 추가
  + 목적지까지의 경로 고정, 경로가 하나인 경우에 사용
  + 소규모 네트워크
  + 대역폭에 부담이 적음
  + 보안 유지
  + 경로에 장애 발생 시 다른 경로로 우회 불가
- 동적 라우팅
  + 네트워크 변경을 자동으로 감지하여 라우팅 테이블 업데이트
  + 장애 발생 시 최적의 경로로 전환
  + 대규모 네트워크

> SSID

- Service Set IDentifier
- 액세스 포인트 고유 이름
- 네트워크 이름, 인증, 암호화, 암호화 키 설정
- 비컨 전송 : 자신의 식별 신호를 네트워크에 있는 모든 기기에 주기적으로 전송
  → 같은 SSID인지 문의 / 응답
  → 인증, 접속 요구
  → 승인
- 채널 : 분할된 주파수 대역
